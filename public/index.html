Текущий статус: ручной режим

Дашборд работает автономно — все данные хранятся локально в браузере. Для подключения автообновления через n8n настройте вебхук-эндпоинты ниже.

1

Google Ads API (OAuth 2.0 + GAQL)

Получите customer_id и refresh_token. Запрос через n8n HTTP Node:

POST https://googleads.googleapis.com/v16/customers/{customer_id}/googleAds:search
Authorization: Bearer {access_token}
{
  "query": "SELECT campaign.name, metrics.impressions, metrics.clicks,
            metrics.cost_micros, metrics.conversions
            FROM campaign WHERE segments.date DURING LAST_30_DAYS"
}
2

Яндекс Директ API (JSON-RPC)

Используйте токен OAuth Яндекса. Метод Reports:

POST https://api.direct.yandex.com/json/v5/reports
Authorization: Bearer {yandex_token}
{
  "params": {
    "SelectionCriteria": {"DateFrom": "2024-01-01", "DateTo": "2024-01-31"},
    "FieldNames": ["Impressions","Clicks","Cost","Ctr","AvgCpc","Conversions"]
  }
}
3

A-ADS API (API Key)

Эндпоинт статистики по кампании:

GET https://a-ads.com/campaigns/{campaign_id}/stats/per_day
?start_date=2024-01-01&end_date=2024-01-31
Authorization: Bearer {api_key}
4

LinkedIn Marketing API

OAuth 2.0, скоуп r_ads_reporting:

GET https://api.linkedin.com/v2/adAnalyticsV2
?q=analytics&dateRange.start.year=2024&dateRange.start.month=1
&pivot=CAMPAIGN&fields=impressions,clicks,costInLocalCurrency,conversions
Authorization: Bearer {linkedin_token}
5

Planfix CRM (REST API)

Считаем лиды за период через фильтр по дате создания:

POST https://api.planfix.com/rest/task/?filters={"dateRange":"last30days"}
X-Planfix-Token: {planfix_token}
6

n8n Webhook → Dashboard

Настройте Schedule Trigger (каждые 6 часов) → HTTP запросы к API → Function Node для форматирования → HTTP POST к дашборду:

POST http://your-dashboard-url/api/update
{
  "channel": "google",
  "period": "2024-01-01/2024-01-31",
  "data": { "budget": 15000, "impressions": 450000, ... }
}
