Google Ads API
Бесплатно REST + OAuth 2.0

Google Ads API даёт доступ к данным кампаний в реальном времени: показы, клики, расходы, конверсии. Данные на дашборд подтягиваются через GAQL (Google Ads Query Language).

Шаг 1 — Включить API доступ
1
Войдите в Google Ads → Инструменты → Центр API
2
Нажмите «Подать заявку на доступ к API». Укажите тип: «Basic access» (достаточно для 15 000 операций/день)
3
Ждите одобрения Google (обычно 1–3 рабочих дня)
Шаг 2 — OAuth 2.0 credentials
1
Откройте Google Cloud Console → создайте проект
2
APIs & Services → Enable APIs → включите «Google Ads API»
3
Credentials → Create credentials → OAuth 2.0 Client ID → Desktop app
4
Скопируйте client_id и client_secret
Шаг 3 — Получить refresh_token
# Установи библиотеку pip install google-ads # Запусти генератор токена python -m google.ads.googleads.oauth2 --client_id=ВАШ_CLIENT_ID --client_secret=ВАШ_SECRET # Откроется браузер → разрешите доступ → скопируйте refresh_token
Шаг 4 — Запрос данных
const query = ` SELECT campaign.name, metrics.impressions, metrics.clicks, metrics.cost_micros, metrics.ctr, metrics.conversions FROM campaign WHERE segments.date DURING LAST_30_DAYS `; const response = await fetch( `https://googleads.googleapis.com/v17/customers/${CUSTOMER_ID}/googleAds:search`, { method: 'POST', headers: { 'Authorization': `Bearer ${access_token}`, 'developer-token': 'ВАШ_DEVELOPER_TOKEN' }, body: JSON.stringify({ query }) } );
Совет: cost_micros нужно делить на 1 000 000 чтобы получить сумму в валюте.
Важно: Developer Token выдаётся отдельно в Центре API Google Ads. Без него запросы не проходят.
Яндекс Директ API
Бесплатно JSON-RPC + OAuth

API Яндекс Директ 5-й версии. Работает через JSON-RPC протокол. Нужен OAuth-токен через Яндекс ID.

Шаг 1 — Создать приложение
1
Откройте oauth.yandex.ru → «Зарегистрировать новое приложение»
2
Название: «BiXBiT Dashboard». Платформа: «Веб-сервисы»
3
Права доступа: Яндекс.Директ → «Управление кампаниями» (read) + «Чтение финансовых данных»
4
Callback URL: https://oauth.yandex.ru/verification_code
5
Скопируйте Client ID
Шаг 2 — Получить токен
# Открой в браузере (замени CLIENT_ID) https://oauth.yandex.ru/authorize?response_type=token&client_id=CLIENT_ID # После авторизации токен будет в URL https://oauth.yandex.ru/verification_code#access_token=ВАШ_ТОКЕН&...
Шаг 3 — Запрос статистики
const response = await fetch('https://api.direct.yandex.com/json/v5/reports', { method: 'POST', headers: { 'Authorization': `Bearer ${TOKEN}`, 'Accept-Language': 'ru', 'returnMoneyInMicros': 'false' }, body: JSON.stringify({ params: { SelectionCriteria: { DateFrom: '2026-05-01', DateTo: '2026-05-31' }, FieldNames: ['Impressions', 'Clicks', 'Cost', 'Ctr', 'Conversions'], ReportName: 'BiXBiT_Report', ReportType: 'CAMPAIGN_PERFORMANCE_REPORT', DateRangeType: 'CUSTOM_DATE', Format: 'TSV', IncludeVAT: 'NO' } }) });
Совет: Отчёты Яндекс Директ генерируются асинхронно. Первый запрос возвращает 201/202, повторный 200 + данные.
A-ADS API
Бесплатно REST + API Key

Самый простой API из всех — достаточно одного API-ключа из личного кабинета. Нет OAuth, нет сложной авторизации.

Шаг 1 — Получить API ключ
1
Войдите на a-ads.com в свой аккаунт
2
Settings → API Key → Generate / Copy
Шаг 2 — Запросить статистику
# Статистика по кампании за период GET https://a-ads.com/api/1/campaign_stats_by_day Параметры: api_key = ВАШ_API_KEY id = ID_КАМПАНИИ date_from = 2026-05-01 date_to = 2026-05-31 # Пример запроса const res = await fetch( `https://a-ads.com/api/1/campaign_stats_by_day?api_key=${KEY}&id=${ID}&date_from=2026-05-01&date_to=2026-05-31` ); const data = await res.json(); // data[].views — показы // data[].clicks — клики // data[].spent — расходы (BTC)
Конвертация BTC → USD: Используй CoinGecko API https://api.coingecko.com/api/v3/simple/price?ids=bitcoin&vs_currencies=usd для текущего курса.
LinkedIn Marketing API
Нужна верификация REST + OAuth 2.0

LinkedIn требует верификации приложения для доступа к Marketing Analytics API. Процесс занимает до 5 рабочих дней.

Шаг 1 — Создать LinkedIn App
1
Откройте linkedin.com/developers/apps → Create App
2
Привяжите к LinkedIn Company Page BiXBiT
3
Products → запросите «Marketing Developer Platform»
4
После одобрения получите разрешения: r_ads r_ads_reporting
Шаг 2 — OAuth + запрос
# Получить access_token через OAuth Authorization Code flow # Запрос аналитики кампаний GET https://api.linkedin.com/rest/adAnalytics Параметры: q=analytics dateRange.start.day=1&dateRange.start.month=5&dateRange.start.year=2026 dateRange.end.day=31&dateRange.end.month=5&dateRange.end.year=2026 pivot=CAMPAIGN fields=impressions,clicks,costInLocalCurrency,conversions Headers: Authorization: Bearer ACCESS_TOKEN LinkedIn-Version: 202502
Стоимость: LinkedIn API бесплатный при использовании для собственных кампаний. Ограничение: 100 запросов в день на токен.

1

2
Скачайте JSON-ключ сервисного аккаунта
3
В GA4: Admin → Property Access Management → добавьте email сервисного аккаунта с ролью Viewer
Шаг 2 — Запрос данных по языковым версиям
const body = { dateRanges: [{ startDate: '2026-05-01', endDate: '2026-05-31' }], dimensions: [{ name: 'pagePath' }, { name: 'country' }], metrics: [ { name: 'sessions' }, { name: 'bounceRate' }, { name: 'averageSessionDuration' }, { name: 'conversions' } ], dimensionFilter: { filter: { fieldName: 'pagePath', stringFilter: { matchType: 'BEGINS_WITH', value: '/ru/' } // /en/ или /es/ } } }; const res = await fetch( `https://analyticsdata.googleapis.com/v1beta/properties/${PROPERTY_ID}:runReport`, { method: 'POST', headers: { Authorization: `Bearer ${token}` }, body: JSON.stringify(body) } );
Google Search Console API
Бесплатно Webmasters v3

Данные о позициях в выдаче, показах и кликах из органического поиска. Работает с теми же OAuth credentials что и GA4.

Запрос позиций по языковым версиям
POST https://www.googleapis.com/webmasters/v3/sites/ https%3A%2F%2Fbixbit.io%2F/searchAnalytics/query { "startDate": "2026-05-01", "endDate": "2026-05-31", "dimensions": ["page", "query"], "dimensionFilterGroups": [{ "filters": [{ "dimension": "page", "operator": "contains", "expression": "/ru/" // или /en/ /es/ }] }], "rowLimit": 10 } // В ответе: clicks, impressions, ctr, position — по каждой странице

Бесплатно Reporting API v1

Тот же OAuth токен что и для Яндекс Директ. Просто добавь права Яндекс.Метрика при создании приложения.

Запрос данных (время на сайте, отказы)
GET https://api-metrika.yandex.net/stat/v1/data Параметры: ids = ID_СЧЁТЧИКА date1 = 2026-05-01 date2 = 2026-05-31 metrics = ym:s:visits,ym:s:bounceRate,ym:s:avgVisitDurationSeconds dimensions = ym:s:URLPathLevel1 // /ru, /en, /es oauth_token= ВАШ_ТОКЕН // Топ страниц: dimensions = ym:s:pageTitle sort = -ym:s:visits limit = 10
Planfix API
Webhook + REST API v2

Для дашборда нужно считать лиды по источнику. Рекомендуемый способ — Webhook при создании задачи/сделки + UTM-метки для определения канала.

Шаг 1 — Webhook при создании лида
1
Планфикс → Настройки → Интеграции → Webhook
2
Событие: «Задача создана» (или «Контакт создан» — зависит от воронки)
3
URL: ваш бэкенд-эндпоинт (или n8n/Make webhook URL)
Шаг 2 — Запрос лидов через API
POST https://your-account.planfix.ru/api/task/list Headers: Authorization: Bearer API_TOKEN Content-Type: application/json Body: { "offset": 0, "pageSize": 100, "filters": [{ "type": 102, // фильтр по дате "operator": "gte", "value": "2026-05-01" }], "fields": "id,name,status,assignees,customFieldData" } // customFieldData — кастомные поля с UTM-меткой источника
Лайфхак: Добавь в Planfix кастомное поле «Источник (UTM)» и заполняй его автоматически через форму на сайте. Тогда сможешь группировать лиды по каналу прямо в API-запросе.
Автоматическое обновление

HTML-файл — статичный. Для автообновления данных нужен один из подходов:

Вариант A — n8n (рекомендуется)

Бесплатный self-hosted инструмент автоматизации. Каждые 15–30 минут опрашивает API и сохраняет результат в JSON-файл, который загружает дашборд.

1
Установите n8n: docker run -p 5678:5678 n8nio/n8n
2
Создайте Workflow: Cron (каждые 15 мин) → HTTP Request (Google Ads API) → Write File (data/google_ads.json)
3
В дашборде замените rnd() на fetch('/data/google_ads.json')
Вариант B — Google Sheets как промежуточный слой

Подключите все API к Google Sheets через Google Apps Script. Sheets обновляется по расписанию, дашборд читает Sheets через Sheets API.

// Apps Script — запускается каждые 30 минут function updateDashboardData() { const token = getGoogleAdsToken(); const data = fetchGoogleAdsData(token); const sheet = SpreadsheetApp.getActiveSheet(); sheet.getRange('A1').setValue(data.clicks); // ... и т.д. }
Вариант C — Make (Integromat)

Облачная альтернатива n8n. Платный, но есть бесплатный план на 1000 операций/месяц. Проще настроить без кода.

Итоговая архитектура: API-источники → n8n/Make каждые 15–30 мин → JSON-файлы на сервере → дашборд fetch() при загрузке страницы. Данные всегда свежие, нет нагрузки на API.
